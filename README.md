# Демонстрационный пешеходный переход на ESP32 и ESP-NOW

## Описание

Проект представляет собой беспроводной демонстрационный пешеходный переход, состоящий из двух автономных стоек:

### Стойка А (Master)

Содержит:

* ESP32
* кнопку вызова пешеходной фазы
* пешеходный светофор
* индикатор "Ждите"
* индикатор "Можно идти"

Стойка управляет всей логикой системы.

### Стойка Б (Slave)

Содержит:

* ESP32
* транспортный светофор

Стойка получает команды от Master через ESP-NOW и отображает транспортные сигналы.

---

# Принцип работы

## Исходное состояние

Транспортный светофор:

```text
Зелёный
```

Пешеходный светофор:

```text
Красный
```

Индикация кнопки:

```text
Выключена
```

---

## Нажатие кнопки

После нажатия кнопки:

Транспортный светофор:

```text
Зелёный
```

Пешеходный светофор:

```text
Красный
```

Индикация:

```text
Ждите
```

Повторные нажатия игнорируются до завершения цикла.

---

## Подготовка остановки транспорта

По истечении времени ожидания:

```cpp
VEHICLE_CLEAR_TIME
```

Транспортный светофор переключается на:

```text
Жёлтый
```

---

## Пешеходная фаза

После завершения жёлтого сигнала:

Транспортный светофор:

```text
Красный
```

Пешеходный светофор:

```text
Зелёный
```

Индикация кнопки:

```text
Можно идти
```

---

## Завершение перехода

По истечении времени разрешающей фазы:

Пешеходный зелёный начинает мигать.

После завершения мигания система возвращается в исходное состояние.

---

# Аварийный режим

Если связь между стойками потеряна более чем на заданный интервал:

Транспортный светофор:

```text
Мигающий жёлтый
```

Пешеходный светофор:

```text
Красный
```

Индикация кнопки:

```text
Выключена
```

После восстановления связи система автоматически возвращается к нормальной работе.

---

# Конечный автомат

Используются следующие состояния:

```cpp
IDLE
REQUESTED
VEHICLE_YELLOW
PEDESTRIAN_GREEN
PEDESTRIAN_FLASH
FAULT
```

---

# Передача данных

Для обмена используется ESP-NOW.

Master периодически отправляет текущее состояние автомата.

Slave отображает соответствующее состояние транспортного светофора.

---

# Настраиваемые параметры

```cpp
const uint32_t VEHICLE_CLEAR_TIME = 5000;
const uint32_t YELLOW_TIME = 3000;
const uint32_t PEDESTRIAN_GREEN_TIME = 10000;
const uint32_t PEDESTRIAN_FLASH_TIME = 5000;
const uint32_t HEARTBEAT_TIMEOUT = 3000;
```

Все значения задаются в миллисекундах.

# Назначение выходов

## Master

```text
Кнопка
Пешеходный красный
Пешеходный зелёный
Индикатор "Ждите"
Индикатор "Можно идти"
```

## Slave

```text
Транспортный красный
Транспортный жёлтый
Транспортный зелёный
```

Фактические GPIO назначаются в начале программы и могут быть изменены под конкретную схему подключения.

# Особенности реализации

* отсутствие роутера и точки доступа;
* мгновенная передача команд через ESP-NOW;
* игнорирование повторных нажатий во время активного цикла;
* автоматический возврат в исходное состояние;
* аварийный режим при потере связи;
* простая адаптация под любые релейные модули.
